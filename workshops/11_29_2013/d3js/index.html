What is D3?

"D3.js is a JavaScript library for manipulating documents based on data. D3 helps you bring data to life using HTML, SVG and CSS. D3â€™s emphasis on web standards gives you the full capabilities of modern browsers without tying yourself to a proprietary framework, combining powerful visualization components and a data-driven approach to DOM manipulation." - D3js.org

D3plus is a library built to take advantage of the robust featureset of D3, while providing a fairly low barrier of entry to users new to code, design, and/or data visualization. Just as it is impossible to tell the story of Brazil without devoting a few lines to Portugal, it is also the case with D3 and D3plus. The project is built entirely dependent on the D3 library and does not intend to duplicate any of its purpose but instead add atop a layer of ease for the user to access its powerful capabilities.

How to use D3

Selecting a DOM element

d3.select() will select a DOM element based on standard selector strings. In this example, we are creating a D3js element from an <svg> on the page with an ID of "viz". You can read more about D3 selections here.

Adding a DOM element to the page

.append() will add a new element to the page. In this example, we are creating a <rect> node to add to the <svg> from the previous example. You can read more about how this works here.

Binding Data to DOM elements

So far everything we've done could also be done via jQuery. .data() enables the binding of data to specific elements on the page, thus driving their appearance or disappearance. In this example, we are binding a simple array of values to our previous <rect> node that is already on the page. You can read more about how the data operator here.

Dynamically Adding Elements New Elements With Data

Now you may be wondering what happens to the other 3 elements in the data array, since their is only 1 <rect> but 4 numbers in our array. Well, D3 was created to handle dynamic changing of data. D3 will apply the first number in our array to our previously build <rect>, and treat the other numbers as "new" rectangles. We can use the .enter() function to tell our page what to do with those new numbers. Here, we will create 3 new rectangles for the rest of the data.

Updating the Rectangles

Great, now we have 4 rectangles on the page. How are we able to modify all of the rectangles, whether they are the new ones or the old ones? It's simple, any styling that we apply to our "rect" variable will effect all active rectangles with data. Here, we set the size of the rectangles based on their data. D3 lets us use the data binded to each element as a variable passed through a function.

Updating the Data

What if we were to update our data? How can we tell our elements on the page about this update and view this change? In this example we change our data to remove one element and update the values of our other elements.

Getting Rid of Old Elements

Now that we've both added and updated our elements to the page, it is time to (sadly) see some go. In the last example we removed one element from our array and as such the corresponding element on the page should also disappear. The example here illustrates this practice via the .exit() method. We select the rectangles that are leaving, and remove them.
